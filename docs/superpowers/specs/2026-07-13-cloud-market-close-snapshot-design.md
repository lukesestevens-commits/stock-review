# 云端收盘市场快照固化设计

## 背景

复盘页的“今日主线 / 强势方向”和“一句话市场判断”目前通过 `/api/market-snapshot` 即时读取公开行情。云端每次请求都重新访问上游数据源，缺少跨设备共享的稳定缓存。上游接口偶发失败或返回降级结果时，这两个字段可能无法自动填写；手机与电脑也无法保证使用同一份收盘数据。

## 目标

- 盘中继续读取当天最新有效市场快照。
- 收盘后以当日最后一份确认有效的快照作为权威结果。
- 把收盘结果按交易日保存到云端，手机与电脑读取同一份数据。
- 自动填写“今日主线 / 强势方向”和“一句话市场判断”。
- 上游接口失败时保留并返回最近一次有效缓存，不让字段变为空白。
- 当日快照固化后不再被后续公开接口结果覆盖。

## 非目标

- 不改变资金、持仓、成交记录和交易评分的同步规则。
- 不要求本地复盘助手在收盘时持续在线。
- 不引入新的第三方行情账号或人工收盘操作。
- 不修改用户手工填写的其他复盘字段。

## 方案比较

### 浏览器本地缓存

实现简单，但缓存只存在当前设备，手机与电脑可能看到不同结果，不能满足云端同步要求。

### 云端按交易日固化（采用）

云端 Worker 获取并校验市场快照，按交易日保存盘中最新值和收盘固化值。所有设备读取同一个云端结果，且不依赖本地电脑持续在线。

### 本地助手收盘上传

可以复用本地抓取能力，但电脑休眠、关闭或助手未运行时会漏掉收盘固化，不适合作为唯一链路。

## 云端数据模型

新增独立的市场快照表，以交易日为主键，保存：

- `trade_date`：行情所属交易日，由指数行情时间确定。
- `updated_at`：快照生成时间。
- `finalized_at`：收盘固化时间；盘中为空。
- `payload_json`：完整市场快照，包括指数、板块、主线、市场判断和数据源质量。

交易快照与同花顺账户快照分表保存，避免市场数据更新影响现有资金与交易同步。

## 收盘判定与固化规则

云端使用 `Asia/Shanghai` 交易时区。市场快照只有同时满足以下条件才固化：

1. `mainLines` 与 `marketOne` 均为非空有效字符串。
2. 三大指数均包含行情时间。
3. 三大指数行情日期一致，且为目标交易日。
4. 三大指数行情时间均不早于当日 `15:00:00`。

盘中快照只作为可更新缓存；首次取得满足上述条件的收盘快照后写入 `finalized_at`。同一交易日已经固化时，云端直接返回固化结果，不再请求上游覆盖。

## 请求与缓存流程

1. 页面加载后立即请求 `/api/market-snapshot`。
2. 云端先读取当日市场快照。
3. 已固化时直接返回数据库内容。
4. 未固化且缓存仍在 60 秒有效期内时返回缓存。
5. 缓存过期时请求公开行情，校验后更新盘中缓存。
6. 新快照满足收盘条件时将其固化。
7. 上游请求失败时返回数据库中的最近一次有效缓存，并标记为陈旧数据；没有任何有效缓存时才返回失败。

页面把市场刷新从三秒成交轮询中拆分，首次立即读取，此后每 60 秒刷新。页面仅在快照同时含有 `mainLines` 和 `marketOne` 时写入表单；收盘固化快照作为这两个字段的最终权威值。

## 错误处理

- 上游接口暂时失败：返回同交易日最近一次有效缓存，不清空页面字段。
- 收盘后取得的行情时间仍早于 15:00：不固化，保留盘中缓存并在下一次刷新重试。
- 板块数据为空或市场判断不完整：拒绝写入和固化，保留旧缓存。
- 数据库暂时不可用：接口返回明确错误；页面保留已有内容。
- 非交易日没有当日行情：不把旧交易日数据错误固化为当天数据。

## 测试策略

- 市场快照判定测试：三大指数收盘时间齐全时可固化，盘中或跨日数据不可固化。
- 云端存储测试：按交易日读写、盘中更新、固化后不可覆盖。
- Worker 测试：盘中缓存、60 秒刷新、收盘固化、上游失败回退和数据库失败响应。
- 页面测试：市场数据使用独立 60 秒定时器；有效快照自动填写两个字段；无效或失败结果不清空旧值。
- 完整回归测试：资金、持仓、真实成交时间和金额格式保持不变。
- 发布后验证：线上接口连续请求返回同一个已固化快照，页面自动显示非空主线和市场判断。

## 验收标准

- 收盘后云端返回当日已固化市场快照。
- “今日主线 / 强势方向”和“一句话市场判断”自动填写且非空。
- 手机与电脑读取到完全一致的字段内容。
- 同一交易日固化后的内容不会继续变化。
- 上游接口故障不会清空已经缓存或已经填写的内容。
- 现有测试与新增回归测试全部通过，线上版本完成验证。

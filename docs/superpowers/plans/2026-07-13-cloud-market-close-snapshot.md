# 云端收盘市场快照固化实施计划

**目标：** 按交易日缓存市场快照，在三大指数确认收盘后固化，并让云端页面稳定自动填写主线和市场判断。

**架构：** 新增独立 D1 市场快照存储。云端 Worker 先读缓存，再按 60 秒新鲜度请求公开行情；满足收盘条件后写入不可覆盖的最终快照。页面使用独立市场定时器读取该接口，只应用字段完整的快照。

## 文件范围

- 新增 `cloud/market-snapshot-store.mjs`：按交易日读写盘中与最终市场快照。
- 新增 `cloud/market-snapshot-policy.mjs`：提取行情交易日、校验字段并判定收盘。
- 新增 `drizzle/0001_market_snapshots.sql`：声明云端市场快照表。
- 新增对应存储与策略测试。
- 修改 `cloud/worker.mjs` 和 Worker 测试：缓存、刷新、固化与失败回退。
- 修改 `tools/build-cloud-site.mjs`：打包新增云端模块。
- 修改 `index.html` 和页面测试：拆分 60 秒市场刷新，只写入完整快照。

## 任务 1：收盘判定策略

1. 先写失败测试，覆盖字段缺失、盘中行情、跨日行情和三大指数均达到 15:00 的情况。
2. 实现纯函数：市场快照有效性、行情交易日提取、收盘判定。
3. 运行策略测试确认通过。

## 任务 2：D1 市场快照存储

1. 先写失败测试，覆盖按日期读取、最新缓存读取和固化后拒绝覆盖。
2. 新增表结构和存储模块。
3. 运行存储测试确认通过。

## 任务 3：Worker 缓存与固化

1. 扩展 Worker 测试，覆盖盘中 60 秒缓存、收盘固化、固化后不再请求上游、上游失败返回旧缓存。
2. 在 `/api/market-snapshot` 接入策略与存储。
3. 返回快照缓存状态、交易日和固化时间，保留现有市场字段结构。
4. 运行 Worker 与云端构建测试。

## 任务 4：页面自动填写

1. 先修改页面测试，要求成交轮询不再请求市场接口，市场接口独立每 60 秒刷新。
2. 仅当 `mainLines` 和 `marketOne` 同时非空时应用快照。
3. 相同 `updatedAt` 不重复写入；无效或失败结果保留页面已有内容。
4. 运行页面与浏览器逻辑测试。
